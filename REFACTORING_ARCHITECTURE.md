# Рефакторенная архитектура Tor HTTP Proxy

## Обзор

Проект был полностью рефакторен с разделением логики между пятью специализированными классами. Каждый класс отвечает за конкретную область функциональности.

## Новая архитектура

### 1. TorConfigBuilder (`src/tor_config_builder.py`)
**Ответственность**: Создание конфигураций Tor

**Функции**:
- Генерация файлов конфигурации `.torrc`
- Создание временных конфигураций
- Поддержка конфигураций с выходными узлами и без них
- Очистка конфигурационных файлов

**Ключевые методы**:
- `create_config_with_exit_nodes()` - создает конфигурацию с указанными выходными узлами
- `create_config_without_exit_nodes()` - создает базовую конфигурацию
- `create_temporary_config()` - создает временный файл конфигурации

### 2. TorProcessManager (`src/tor_process_manager.py`)
**Ответственность**: Управление одним процессом Tor и мониторинг его здоровья

**Функции**:
- Запуск и остановка процесса Tor
- Мониторинг здоровья процесса
- **Проверка выходного IP каждые 5 секунд**
- Сбор статистики об активности выходных узлов
- Перезагрузка выходных узлов без перезапуска процесса

**Ключевые методы**:
- `start()` - запускает процесс и начинает мониторинг
- `check_health()` - проверяет здоровье процесса
- `get_status()` - возвращает текущий статус
- `reload_exit_nodes()` - перезагружает список выходных узлов

### 3. TorPoolManager (`src/new_tor_pool_manager.py`)
**Ответственность**: Параллельный запуск множества процессов Tor

**Функции**:
- **Максимум 20 одновременных процессов** для избежания конкуренции за ресурсы
- Параллельный запуск и остановка процессов
- Мониторинг состояния всех процессов в пуле
- Перезапуск неудачных процессов
- Использует `TorProcessManager` для управления отдельными процессами

**Ключевые методы**:
- `start_processes()` - запускает множество процессов параллельно
- `check_process_health()` - проверяет здоровье конкретного процесса
- `restart_failed_processes()` - перезапускает неудачные процессы

### 4. ExitNodeValidator (`src/exit_node_validator.py`)
**Ответственность**: Проверка выходных нод на пригодность для парсинга

**Функции**:
- **Отправляет 6 запросов** на страницу Steam для каждой ноды
- **Считает ноду валидной если 3 или больше запросов успешны**
- Параллельная проверка множества нод
- Использует временные Tor процессы для тестирования

**Ключевые методы**:
- `validate_exit_nodes()` - проверяет список выходных нод
- `get_validation_stats()` - возвращает статистику валидации

**Конфигурация**:
- URL для тестирования: `https://steamcommunity.com/market/search?appid=730`
- Количество запросов на ноду: 6
- Минимум успешных запросов: 3

### 5. TorOrchestrator (`src/tor_orchestrator.py`)
**Ответственность**: Координация всей системы

**Функции**:
- Получение выходных нод от `TorRelayManager`
- Валидация нод через `ExitNodeValidator`
- Запуск процессов через `TorPoolManager`
- Распределение валидных нод по процессам
- Добавление процессов в `HTTPLoadBalancer`
- Мониторинг и обслуживание системы

**Ключевые методы**:
- `start_system()` - запускает всю систему
- `stop_system()` - останавливает всю систему
- `get_system_status()` - возвращает полный статус
- `restart_failed_processes()` - перезапускает неудачные процессы

## Процесс запуска системы

1. **Запуск балансировщика** - `HTTPLoadBalancer` начинает слушать HTTP запросы
2. **Получение выходных нод** - `TorRelayManager` загружает список доступных нод
3. **Валидация нод** - `ExitNodeValidator` проверяет ноды запросами к Steam
4. **Распределение нод** - валидные ноды распределяются между процессами
5. **Запуск процессов** - `TorPoolManager` запускает процессы Tor параллельно
6. **Добавление в балансировщик** - успешные процессы добавляются в балансировщик
7. **Мониторинг** - система начинает автоматический мониторинг и обслуживание

## Использование

### Запуск с новой архитектурой

```bash
# Использование нового main файла
python src/new_main.py

# Конфигурация через переменные окружения
TOR_PROCESSES=15 LISTEN_PORT=8081 VALIDATE_NODES=true python src/new_main.py
```

### Переменные окружения

- `TOR_PROCESSES` - количество процессов Tor (по умолчанию: 20, максимум: 20)
- `LISTEN_PORT` - порт HTTP прокси (по умолчанию: 8081)
- `VALIDATE_NODES` - включить валидацию нод (по умолчанию: true)

### Программное использование

```python
from tor_orchestrator import TorOrchestrator

# Создание оркестратора
orchestrator = TorOrchestrator(listen_port=8081)

# Запуск системы с 15 процессами и валидацией нод
success = orchestrator.start_system(process_count=15, validate_nodes=True)

if success:
    print("System started successfully!")
    
    # Получение статуса
    status = orchestrator.get_system_status()
    print(f"Active processes: {status['active_processes']}")
    
    # Перезапуск неудачных процессов
    result = orchestrator.restart_failed_processes()
    print(f"Restarted: {len(result['successful'])} processes")

# Остановка системы
orchestrator.stop_system()
```

## Мониторинг

### Проверка выходного IP
Каждый процесс проверяет свой выходной IP каждые 5 секунд и обновляет статистику активности узлов.

### Статистика системы
Оркестратор предоставляет детальную статистику:
- Общее количество процессов
- Активные процессы
- Валидированные ноды
- Статистика балансировщика
- Индивидуальные статусы процессов

### Автоматическое обслуживание
- Проверка здоровья процессов каждые 2 минуты
- Автоматический перезапуск неудачных процессов каждые 5 минут
- Удаление нездоровых процессов из балансировщика

## Преимущества новой архитектуры

### Разделение ответственности
- Каждый класс имеет четко определенную роль
- Упрощенное тестирование и отладка
- Легкость добавления новой функциональности

### Масштабируемость
- Ограничение в 20 одновременных процессов предотвращает перегрузку
- Параллельное выполнение операций
- Эффективное использование ресурсов

### Надежность
- Детальный мониторинг каждого компонента
- Автоматическое восстановление от сбоев
- Graceful shutdown всей системы

### Производительность
- Проверка IP каждые 5 секунд обеспечивает актуальную информацию
- Валидация нод гарантирует работоспособность для Steam
- Эффективное распределение нагрузки
